Instalación

¿Qué es KNIME?

KNIME (Konstanz Information Miner) es una plataforma de análisis de datos de código abierto que permite crear flujos de trabajo visuales arrastrando y conectando nodos, sin necesidad de programar (aunque también lo permite). Se utiliza para tareas como limpieza de datos, integración, análisis, machine learning y visualización, facilitando el trabajo con grandes volúmenes de información de forma intuitiva y modular.

1. Descargar el instalador

Primero, buscamos en el navegador "knime analytics platform" y hacemos clic en el primer resultado. Al hacerlo, se nos abrirá la página web oficial de KNIME.

Una vez estemos en esta página, damos clic en el botón que dice "Download KNIME for free", lo que nos llevará a la siguiente página.

Para que nos permita descargar el instalador, primero se requiere digitar un correo electrónico; una vez se digite, aceptamos los términos y condiciones y damos clic en "Download".

Por último, podremos descargar el instalador para el sistema operativo de nuestra preferencia. En este caso, vamos a usar Windows, así que descargamos el que dice "KNIME Analytics Platform for Windows (Installer)".

2. Proceso de instalación

1. Elegimos el modo de instalación; en este caso, seleccionamos la opción recomendada "Install for me only".

2. Leemos el acuerdo de licencia, aceptamos los términos y condiciones y hacemos clic en "Next".

3. Definimos en qué carpeta se va a instalar; dejamos la que se crea por defecto y hacemos clic en "Next".

4. Definimos el nombre acortado de la aplicación; dejamos el que se crea por defecto y hacemos clic en "Next".

5. Seleccionamos las configuraciones adicionales que queramos; por defecto aparecen todas seleccionadas, vamos a dejarlo así y hacemos clic en "Next".

6. Especificamos la cantidad de memoria RAM que se reservará para la aplicación y hacemos clic en "Next".

7. Por último, revisamos que todas las configuraciones hayan quedado bien y hacemos clic en "Install".

8. Una vez haya finalizado el proceso de instalación, nos permitirá abrir la aplicación cuando hagamos clic en "Finish".

9. La primera vez que se abra la aplicación, nos indicará en qué carpeta se destinarán los workflows y otros archivos; dejamos la que se crea por defecto y seleccionamos "Use this as the default and do not ask again" para evitar que aparezca esa ventana nuevamente, y hacemos clic en "Launch".

ETL Básico

CSV Reader

Con este nodo se nos permite leer datos desde un archivo de tipo CSV. Este archivo contiene información sobre la relación entre los datos de clientes, productos y proveedores, y tiene las siguientes columnas: ID_Cliente, Nombre, Edad, Genero, Producto, Categoria, Cantidad, Precio, Sede, Proveedor, Fecha_Compra, Comentario, Codigo_Interno.

Column Filter

Este nodo se conecta a la salida del CSV Reader y nos permite excluir las columnas que no necesitemos dentro de nuestro análisis; para este caso, excluimos las columnas de nombre, comentario y codigo_interno.

Missing Value

Este nodo se conecta a la salida del Column Filter y nos ayuda a rellenar los campos vacíos dentro del CSV; los campos catalogados como numéricos se llenan con el valor medio de los datos de su columna, y los de tipo String (texto) se llenan con el valor más frecuente de su columna.

Rule Engine

Este nodo se conecta a la salida del Missing Value y nos permite generar una nueva columna a partir de una condición; en este caso, la condición se basa en la edad y valida si el cliente es menor, adulto o adulto mayor, según un rango de edades.

Statistics

Este nodo se conecta a la salida del Rule Engine y nos muestra una tabla con estadísticas obtenidas de todas nuestras columnas y registros, como: el valor mínimo, valor máximo, valor medio, etc.

Histogram

Este nodo se conecta a la salida de ocurrencia del Statistics y nos permite elegir una variable para que la grafique en un histograma; en este caso, se eligió la variable de precio y nos indica cuántos productos hay en relación con un precio en específico.

Scatter Plot

Este nodo se conecta a la salida de ocurrencia del Statistics y nos permite comparar entre dos variables dentro de un diagrama de dispersión; en este caso, elegimos Precio y Edad, lo que nos indica la edad de los clientes que compran un producto más caro o más barato.

Nodos

1. File Reader

Este nodo permite leer archivos de texto como .txt o datos delimitados que no necesariamente son CSV. Sirve para importar información desde distintas fuentes y configurar cómo se separan los datos (por comas, tabulaciones, etc.), siendo útil cuando se trabaja con formatos menos estructurados.

Se elige el archivo de texto y se define el delimitador de columna correspondiente al utilizado en el archivo, para separar cada columna.

2. Joiner

El Joiner une dos tablas usando una columna en común, similar a los JOIN en SQL. Sirve para combinar información de diferentes fuentes (por ejemplo, clientes y compras) y se configura seleccionando las columnas clave y el tipo de unión (inner, left, right, full).

Para este nodo se necesitan dos archivos de datos que guarden relación entre sí, y se define qué variable de cada tabla se utilizará para mostrar los registros que tengan coincidencia.

Se puede elegir si los datos deben estar en ambas tablas, si se le da preferencia a la tabla de la derecha o a la de la izquierda, o si se desean incluir todos los datos de ambas tablas.

3. Math Formula

Este nodo permite crear nuevas columnas a partir de operaciones matemáticas entre otras columnas. Sirve para calcular valores como totales, promedios o indicadores, y se configura escribiendo una fórmula usando los nombres de las columnas.

Se multiplica la cantidad por el precio y se agrega una columna llamada total con el resultado de la operación.

4. Row Filter

El Row Filter filtra filas según condiciones específicas, como rangos o comparaciones. Sirve para trabajar solo con los datos relevantes (por ejemplo, clientes mayores de cierta edad) y se configura definiendo la columna y la condición.

Filtra los registros en los que la edad sea mayor a 18.

5. Sorter

El Sorter organiza los datos en orden ascendente o descendente según una o más columnas. Sirve para analizar datos de forma más clara o priorizar información, como ordenar por precio o cantidad.

Ordena los registros por precio, de mayor a menor.

6. GroupBy

Este nodo agrupa datos por una o varias columnas y aplica funciones como suma, promedio o conteo. Sirve para generar resúmenes, similar a tablas dinámicas, y se configura eligiendo columnas de agrupación y agregaciones.

Agrupa la columna de precio junto con la cantidad del producto.

7. Row Splitter

El Row Splitter divide los datos en dos conjuntos según una condición lógica. Sirve para separar información (por ejemplo, mayores y menores de edad) y genera dos salidas: una que cumple la condición y otra que no.

Muestra una tabla con los registros que coinciden con el filtro de que la categoría sea igual a "Oficina" y, en otra tabla, los registros que no coinciden.

8. String Manipulation

Este nodo permite modificar textos mediante funciones (mayúsculas, concatenación, reemplazos, etc.). Sirve para limpiar y transformar datos de tipo string y se configura escribiendo expresiones con funciones predefinidas.

Reemplaza la columna sede con los nombres de las sedes en mayúscula.

9. Number to String

Convierte valores numéricos en texto. Sirve cuando se necesita tratar números como categorías o etiquetas, y se configura seleccionando las columnas a convertir.

Convierte a texto las columnas id_cliente y edad, excluyendo cantidad y precio, ya que estas sí necesitan realizar operaciones.

10. String to Number

Convierte texto a valores numéricos. Sirve cuando los datos fueron leídos incorrectamente como texto y se necesita hacer cálculos con ellos, configurando el tipo de número destino.

En este caso, el sistema identifica correctamente los campos que necesitan ser numéricos, por lo que no es necesario usarlo; sin embargo, es muy útil cuando el programa no interpreta correctamente los tipos.

11. Bar Chart

Este nodo crea gráficos de barras para visualizar datos categóricos. Sirve para comparar cantidades entre categorías (como productos o sedes) y se configura seleccionando columnas de categoría y valor.

Primero seleccionamos la dimensión de la categoría y podemos incluir columnas con valores numéricos para realizar el respectivo análisis; en este caso, es la relación que tienen los nombres con los precios que cada cliente está dispuesto a pagar por un producto.

12. Line Plot

Genera gráficos de líneas para mostrar tendencias a lo largo del tiempo o secuencias. Sirve para analizar evolución (por ejemplo, ventas por fecha) y se configura con ejes X e Y.

Funciona de manera similar al nodo de Bar Chart y, en este caso, se utiliza para ver las tendencias en el tiempo de las ventas realizadas cada día.

13. Pivot

El Pivot transforma filas en columnas, similar a una tabla dinámica. Sirve para reorganizar datos y facilitar análisis comparativos, configurando columnas de agrupación, pivote y agregación.

Primero incluimos las columnas que queramos que sean agrupadas; en este caso, se escogió sede.

En Pivots, convierte los registros, en este caso de producto, en columnas independientes para cada producto.

En Manual Aggregation se escoge la columna de cantidad para que, al final, cree una tabla que indique cuántos productos de un tipo específico hay en cada sede.

14. Duplicate Row Filter

Este nodo elimina filas duplicadas según columnas seleccionadas. Sirve para limpiar datos repetidos y mejorar la calidad del dataset, configurando las columnas clave para detectar duplicados.

Elegimos las columnas que queramos para que, en conjunto, no tengan registros duplicados.

15. Decision Tree Learner

Crea un modelo de árbol de decisión para clasificación o predicción. Sirve para análisis de machine learning básico y se configura definiendo la variable objetivo y las variables predictoras.

Este nodo, a partir de la variable objetivo que es categoria, comienza a generar reglas utilizando las demás columnas, lo que permite mostrar diferentes perspectivas de los mismos datos.
